Sandwich-anator Programming Assignment

PROBLEM DESCRIPTION:

Create a simulation of an automatic sandwich dispensing machine, called the Sandwich-anator.  The machine maintains an inventory of ingredients, and is able to dispense a fixed set of possible sandwiches by combining these ingredients in different amounts. The cost of a sandwich is determined by its component ingredients.

When complete, zip the project (minus .class files) and email it to matt.kasubaski at cdw.com with your name and Sandwich-Anator in the subject.

Upon startup, the Sandwich-anator should display a list of its current inventory, followed by a menu to allow the user to select a sandwich.  As sandwiches are dispensed, the inventory should be updated.  Only sandwiches for which there is sufficient inventory can be served.

The specified input and output formats for the Sandwich-anator must be followed exactly.  At the end of these instructions, you will find examples of some input/output scenarios.

INPUT FORMAT:

Your solution should read from the standard input stream, one command per line.  No prompts or other extraneous user messages should be displayed.  Blank input lines should be ignored.

Each valid command consists of a single character, as follows:
  • 'R' or 'r' - restock the inventory and redisplay the menu
  • 'Q' or 'q' - quit the application
  • [1-7] - order the sandwich with the corresponding number in the menu
If the user enters an invalid command, then the program should display a single-line message with the following format:
Invalid selection: <characters that were entered>
If the user selects a valid sandwich number, and the machine has the required ingredients to make the sandwich, then the program should display a single-line message with the following format:
Dispensing: <sandwich name>
On the other hand, if the sandwich order cannot be completed, then the program should display a single-line message with the following format:
Out of stock: <sandwich name>
The inventory and menu (see next section) should be displayed immediately following any applicable message.

OUTPUT FORMAT:

All output should be written to the standard output stream.  At program startup, and following the processing of every command, the machine inventory and the sandwich menu should be displayed.  Both the inventory list and the sandwich menu should be displayed in alphabetic order (by ingredient name or sandwich name, respectively), in the following format:
Inventory:
<ingredient name>,<quantity in inventory>
...
<ingredient name>,<quantity in inventory>
Menu:
<sandwich number>,<sandwich name>,<cost>,<in-stock>
...
<sandwich number>,<sandwich name>,<cost>,<in-stock>
Sandwiches should be numbered sequentially, starting at 1, in the order they are displayed in the menu. The in-stock indicator should be either "true" or "false".

Note: the sample output is indented in these instructions to make it easier to read.  The output generated by your program should not have any whitespace at the beginning of a line.

TECHNICAL NOTES:

Your solution should be a command-line program written in Java and be buildable by Maven. If you use any external libraries that are not in the Maven Central Repository please bundle these libraries with your code so that we can run your solution.

It is not required that the initial machine configuration (inventory counts, available sandwiches and prices, etc.) be dynamic. In particular, it is acceptable to perform this initialization in code, rather than reading the configuration from an external file or database. However, your program should be flexible enough to allow new sandwiches to be added to the menu without requiring extensive code changes.

Make sure your program works correctly for all combinations of inputs.  You may include automated tests if you like (using a framework such as JUnit), but this is not required.

Extensive inline or method-level comments are not required, unless you want to include them to highlight particular aspects of your design or implementation.
